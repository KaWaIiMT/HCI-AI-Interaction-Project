## 一、选题决策

### 题目选择：AI智能交互（题目8）

**决策依据：**
- 除AI智能交互外的题目（语音SDK/Kinect/Hololens/全景图/VR等）依赖老旧SDK或特殊硬件，调试周期长风险高
- AI交互是当前热点，Claude Code在此领域发挥空间最大
- 评委看"表面丰富度"和"视觉冲击力"胜过技术深度
- 不需要特殊硬件即可开发演示

### 基础项目选择：Super Agent Party（已更名）

**选择理由：**
- 功能覆盖面广：3D虚拟角色 + 语音对话 + 知识库RAG + 多模态交互
- 技术栈现代：Electron + Vue 3 + Python FastAPI + WebSocket
- 代码开源（AGPL-3.0），有完整社区生态
- 功能模块丰富但可裁剪，适合"看起来内容多"

### 3.2 UI改造（轻量改皮）

**改造内容：**
1. 配色方案：从原项目的粉紫色 → 科技蓝+深灰
2. Logo/图标：替换为小组自定义图标
3. 侧边栏菜单：重新排序，移除不需要的功能入口
4. 移除UI上暴露的功能：
   - 酒馆模式（多角色扮演）
   - 直播推流入口
   - 扩展商店入口
   - 即时通讯Bot配置入口
5. 聊天界面保留，调整布局细节

**不改动：**
- 底层 Vue 组件架构
- 核心聊天逻辑
- VRM 渲染管线

### 3.3 功能模块决策

| 模块 | 决策 | 理由 |
|------|------|------|
| LLM对话核心 | ✅ 保留 | 核心功能 |
| VRM 3D桌面宠物 | ✅ 保留 | 视觉冲击力最强 |
| 语音识别/合成 | ✅ 保留 | 核心交互方式 |
| 知识库RAG | ✅ 保留 | 答辩亮点 |
| 即时通讯Bot（QQ/微信/钉钉/飞书/Discord/Slack/Telegram） | ❌ 移除UI入口 | 老师不理解，增加答辩风险 |
| 直播推流（B站/YouTube/Twitch） | ❌ 移除UI入口 | 与课程无关 |
| 酒馆模式（多角色扮演） | ❌ 移除UI入口 | 老师不理解 |
| ComfyUI 工具 | ❌ 移除UI入口 | 非核心 |
| 日记系统 | ⬜ 保留后端，隐藏UI | 可作为"记忆系统"包装 |
| 情感/行为引擎 | ⬜ 保留后端 | 让VRM角色有自然行为 |
| 扩展系统 | ❌ 移除UI入口 | 非必要 |
| 浏览器远程控制 | ❌ 移除UI入口 | 有安全隐患 |
| Dify/Claude/Gemini 桥接 | ⬜ 保留后端 | 多模型支持是加分项 |

## 四、技术架构

├── 前端（Electron + Vue 3）

│   ├── py/agent.py            — 对话Agent

    └── docs/

## 七、决策要点备忘

1. **老师不深问技术** → 代码改造以"面子工程"为主，底层逻辑不动
2. **评委看表面丰富度** → 保留所有看起来"多"的功能模块，只隐藏不删除
3. **答辩占40%** → 优先保证演示流畅、视觉效果炸裂
4. **项目必须有"我们的故事"** → 去品牌化是第一步，后续要在报告里讲"我们设计的交互理念"
